# Командна робота і план

## Приклад розподілу ролей

### Учасник 1

- Базова логіка поля.
- Розміщення мін.
- Перевірка перемоги.
- Написання unit-тестів для логіки.

### Учасник 2

- Інтерфейс на `pygame`.
- Обробка подій миші та клавіатури.
- Повідомлення про стан гри.
- README, UML-діаграми, підготовка до захисту.

## План виконання

1. Створити репозиторій і налаштувати доступи.
2. Створити окремі гілки для кожного учасника.
3. Реалізувати логіку гри.
4. Реалізувати інтерфейс користувача.
5. Додати тести і документацію.
6. Провести злиття гілок.
7. Перевірити запуск і підготувати демонстрацію.

## Git-правила для команди

- Кожен учасник працює лише у власній гілці.
- Кожна завершена підзадача оформлюється окремим комітом.
- Перед merge потрібно оновити свою гілку від `main`.
- Конфлікти вирішуються спільно з коротким описом у pull request.
